この百科事典サイトでは、Wikipediaや萌娘百科で使用されているWikitext構文ではなく、Markdown形式を用いて記事を作成します。

そのため、本ページでは簡易的なMarkdown構文、より正確には、本サイトで記事を作成するために必要十分なMarkdown構文を説明します。

なお、すべての構文記号には半角記号を使用してください。日本語や中国語の入力システムで入力した全角記号は、Markdownの構文として機能しません。

## 見出し

見出しには `#` 記号を使用します。`#` の数が見出しの階層に対応します。

たとえば、`#` はレベル1の見出し、`##` はレベル2の見出しとなり、同様に最大でレベル6まで使用できます。

見出し構文を有効にするには、`#` の後ろに半角スペースを入れる必要があります。

例：

`# レベル1の見出し`

次のように表示されます。

# レベル1の見出し

## テキストの書式

- 太字：**太字テキスト**

  2つのアスタリスク、または2つのアンダースコアを使用します。構文は `**太字にするテキスト**` または `__太字にするテキスト__` です。

- 斜体：*斜体テキスト* または _斜体テキスト_

  1つのアスタリスク、または1つのアンダースコアを使用します。構文は `*斜体にするテキスト*` または `_斜体にするテキスト_` です。

- 太字斜体：***太字斜体テキスト***

  3つのアスタリスクを使用します。構文は `***太字斜体にするテキスト***` です。

- 取り消し線：~~取り消し線テキスト~~

  2つのチルダを使用します。構文は `~~取り消し線を付けるテキスト~~` です。

- インラインコード：バッククォートで対象を囲みます。例：`` `インラインコード` ``

## リスト

### 箇条書きリスト

`-` または `+` を使用します。

- 項目1
- 項目2

構文は次のとおりです。

```
- リストのテキスト
```

リスト記号の後ろには、必ず半角スペースを入れてください。

### 番号付きリスト

数字の後ろにピリオドを付けます。

1. 手順1
2. 手順2
3. 手順3

## ハイパーリンク

リンクの構文は次のとおりです。

```
[リンク名](URL)
```

例：

```
[本サイト](https://kamitsubaki.wiki/ja/)
```

次のように表示されます。

[本サイト](https://kamitsubaki.wiki/ja/)

## 表

`|` で列を区切り、`-` で見出し行との区切りを定義します。

```
| アーティスト | 楽曲名 | 歌詞 |
| :--- | :---: | ---: |
| KAF | 糸 | 省略 |
| RIM | 1999 | 省略 |
```

次のように表示されます。

| アーティスト | 楽曲名 | 歌詞 |
| :--- | :---: | ---: |
| KAF | 糸 | 省略 |
| RIM | 1999 | 省略 |

配置方法：

- `:---`：左揃え
- `:---:`：中央揃え
- `---:`：右揃え

### Frontmatter

ファイル上部のFrontmatterには、編集する記事の属性を記述します。

Frontmatterの開始記号と終了記号には、どちらも `---` を使用します。

例：

```yaml
---
locale: ja
translationKey: example-entry
title: 記事の例
---
```

### Markdownエディターについて

Markdown形式のファイルを作成するために、特別なエディターは必要ありません。

メモ帳などの基本的なテキストエディターでも、保存時に拡張子を `.md` に変更すればMarkdownファイルを作成できます。

Markdownに慣れていない方には、リアルタイムプレビューに対応したエディターの方が使いやすい場合があります。

本ページの作成者は、機能が充実しており、複数のプラットフォームに対応しているObsidianを推奨します。

## 応用的な内容

基本的なMarkdown構文を習得した後は、より高度な表現方法であるHTML構文についても学ぶことができます。

MarkdownではHTMLを利用できるため、HTMLを組み合わせることで、より多様な表示形式を実現できます。

以下では、本サイトの記事編集で比較的よく使用するHTML構文を紹介します。

### Rubyタグ

Rubyは、漢字文化圏で広く使用されている組版形式です。漢字などの本文に、読み方や注釈を添えるために使用します。

たとえば、次のように記述します。

```html
<ruby>局部壊死<rt>きょくぶえし</rt></ruby>
```

文字ごとに読みを正確に対応させる場合は、次のように記述します。

```html
<ruby>観<rt>かん</rt>測<rt>そく</rt>所<rt>じょ</rt></ruby>
```

表示結果：

- <ruby>観<rt>かん</rt>測<rt>そく</rt>所<rt>じょ</rt></ruby>

### 黒幕・秘匿表示の効果

本サイトでは、次のコードを使用して、文字をぼかして隠す効果を実装しています。

```html
<span style="filter: blur(5px); transition: filter 0.3s; cursor: pointer;" onmouseover="this.style.filter='none'" onmouseout="this.style.filter='blur(5px)'" onclick="this.style.filter='none'">これは精神汚染を引き起こす機密文書です。</span>
```

表示結果：

<span style="filter: blur(5px); transition: filter 0.3s; cursor: pointer;" onmouseover="this.style.filter='none'" onmouseout="this.style.filter='blur(5px)'" onclick="this.style.filter='none'">私の姿は見えません。見えません。</span>

使用する際は、`<span>` 要素内の文章を、隠したい内容に置き換えてください。

### 折りたたみと展開

項目が多すぎる場合、特に表の中に多数の内容がある場合は、次の形式を使用して一部の内容を初期状態で非表示にできます。

```html
<details>
  <summary>通常時に表示する見出しをここに記述します。クリックすると展開されます。</summary>
  通常時には非表示にする内容をここに記述します。展開後に表示されます。
</details>
```

表示結果：

<details>
  <summary>通常時に表示する見出しをここに記述します。クリックすると展開されます。</summary>
  通常時には非表示にする内容をここに記述します。展開後に表示されます。
</details>

*注：編集中に改行を挿入する場合は、`<br>` を使用できます。*

### 音声・動画の埋め込み

本サイトでは、`iframe` 形式による動画および音声の埋め込みに対応しています。

主要な動画・音楽配信サービスでは、iframe形式の埋め込みコードを直接生成できます。生成されたコードを記事内にそのまま貼り付けて使用してください。

以上が、本サイトの記事編集で比較的よく使用する構文です。この文書は今後更新される場合があります。

## 属性ブロックガイド

記事を編集する際に、Frontmatterの属性が何を意味するのか分からない場合は、以下の説明を参照してください。

### 共通部分

以下の属性は、すべての記事カテゴリーで共通して使用されます。

- `locale`：文書の言語版を示します。`zh` は中国語、`en` は英語、`ja` は日本語を表します。編集している記事の言語に対応する値を入力してください。
- `translationKey`：同一記事の多言語版を関連付ける共通識別子です。同じ記事の中国語・日本語・英語ファイルには、同一の値を設定してください。

### アーティスト部分

| 属性 | 型 | 必須 | 役割と入力内容 |
| :---: | :---: | :---: | :--- |
| `locale` | `zh / ja / en` | はい | 現在の記事の言語 |
| `translationKey` | 文字列 | はい | 同一人物の各言語版で共通して使用する識別子 |
| `code` | 文字列 | いいえ | 人物番号、資料番号、または内部コード |
| `name` | 文字列 | はい | 現在の言語で表示する人物名 |
| `romanizedName` | 文字列 | はい | ローマ字名、ラテン文字名、または国際表示名 |
| `categoryTitle` | 文字列 | いいえ | 所属カテゴリーのメインタイトル |
| `categorySubtitle` | 文字列 | いいえ | 所属カテゴリーのサブタイトル、または英語による説明 |
| `categoryOrder` | 数値 | いいえ | カテゴリー間の並び順。通常は小さい値ほど先に表示されます |
| `itemOrder` | 数値 | いいえ | 現在の人物を所属カテゴリー内で並べるための値 |
| `meta` | 文字列 | いいえ | 一覧カードに表示する短い補足情報。役割、所属、短い概要などを記述します |
| `debutDate` | 文字列 | いいえ | デビュー日。`YYYY-MM-DD` 形式を推奨しますが、Schemaでは強制されません |
| `profileTagline` | 文字列 | いいえ | 人物詳細ページに表示する紹介用の短い文 |
| `designCredits` | 文字列配列 | いいえ | キャラクターデザイン、ビジュアルデザイン、モデリングなどを担当した制作者の一覧 |
| `affiliations` | 文字列配列 | いいえ | 所属レーベル、グループ、企画、または組織 |
| `officialLinks` | オブジェクト配列 | いいえ | 公式サイトおよび公式SNSへのリンク |
| `officialLinks[].label` | 文字列 | はい | `Official Site`、`YouTube` などのリンク名 |
| `officialLinks[].href` | 文字列 | はい | 公式リンクのURL |
| `featuredEntries` | オブジェクト配列 | いいえ | 人物ページで重点的に関連付ける他の記事 |
| `featuredEntries[].label` | 文字列 | はい | 関連コンテンツの表示名 |
| `featuredEntries[].href` | 文字列 | はい | 関連記事へのパス |
| `featuredEntries[].kind` | 固定列挙値 | はい | 関連コンテンツの種類。`artist`、`project`、`album`、`song` のいずれかのみ使用できます |
| `theme` | 共通テーマオブジェクト | いいえ | 人物詳細ページで使用する個別の配色 |
| `statusLabel` | 文字列 | はい | 「活動状況」など、状態欄に表示する見出し |
| `status` | 文字列 | はい | 「活動中」「活動終了」などの実際の状態 |
| `inactive` | ブール値 | いいえ | 非活動状態かどうか。通常、`true` は活動終了済み、またはアーカイブ済みであることを示します |
| `image` | 文字列 | はい | 人物のメイン画像、アイコン、または立ち絵へのパス |
| `seo` | 共通SEOオブジェクト | いいえ | 現在の記事の検索エンジンおよび共有用情報 |
